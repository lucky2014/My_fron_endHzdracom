jQuery插件开发过程

  1. 定义作用域：为插件定义私有作用域，外部代码不能直接访问插件内部的代码，插件内部的代码不污染全局变量。

    (function($){
    //代码
})(jQuery);

  2. 为jQuery扩展一个插件：为jQuery的实例添加一个扩展方法。该扩展方法可以接收一些参数。

    (function($){
    //代码
    $.fn.myTable = function(options){ //方法名要有实际的意义，英文名要通俗易懂
         //具体实现的代码
    }
})(jQuery);

  3. 设置默认值：为扩展方法设置默认值，一般会将默认属性对象定义为defaults。使用$.extend(defaults,options)将默认值和传入的参数进行合并。

    (function($){
    //代码

    var defaults = {
        len:10,
        index: 1
    };

    $.fn.myTable = function(options){
         //具体实现的代码
         var options = $.extend({},defaults,options);
    }
})(jQuery);

  4. 支持jQuery的连接调用：循环把每个元素返回。

    (function($){
    //代码

    var defaults = {
        len:10,
        index: 1
    };

    $.fn.myTable = function(options){
         //具体实现的代码
         var options = $.extend({},defaults,options);

         return this.each(function(){
             //对每个options的操作
         });
    }
})(jQuery);

  5. 插件里的方法：为了使代码结构化可视化，需要用到function。在插件里的方法不能被外界调用。

    (function($){
    //代码

    var defaults = {
        len:10,
        index: 1,
        name: "name",
        obj: "#id"
    };

    //插件里的方法
    var showName = function(defaults){
        $(defaults.obj).append(function(){
            return defaults.name;
        });
    };

    $.fn.myTable = function(options){
         //具体实现的代码
         var options = $.extend({},defaults,options);

         return this.each(function(){
             //对每个options的操作
             showName(options);
         });
    }
})(jQuery);

  6. 自定义回调函数

    (function($){
    //代码

    var defaults = {
        len:10,
        index: 1,
        name: "name",
        callback: function(){}  
    };

    //插件里的方法
    var showName = function(obj){
        $(obj).append(function(name){
            return name;
        });
    };

    $.fn.myTable = function(options){
         //具体实现的代码
         var options = $.extend({},defaults,options);

         return this.each(function(){
             //对每个options的操作
             showName(options.name);

             //定义回调函数	
             if(options.callback && 'function' == typeof options.callback){
                 callback(options);
             }
         });
    }
})(jQuery);

JQuery的extend函数分析

  1. 类级别（$.extend）,$.extend()，是扩展的jQuery这个类。假设我们把jQuery这个类看成是人类，能吃饭能喝水能跑能跳，现在我们用jQuery.extend这个方法给这个类拓展一个能唱歌的技能。

    • 扩展类

    Jquery的扩展方法extend是我们在写插件的过程中常用的方法。类级别你可以理解为拓展jquery类，使用$.extend方法开发。

    extend(dest,src1,src2,src3...);

    它的含义是将src1,src2,src3...合并到dest中,返回值为合并后的dest,由此可以看出该方法合并后，是修改了dest的结构的。如果想要得到合并的结果却又不想修改dest的结构，可以如下使用：

    var newSrc=$.extend({},src1,src2,src3...)//也就是将"{}"作为dest参数。

    这样就可以将src1,src2,src3...进行合并，然后将合并结果返回给newSrc了。如下例：

    var result=$.extend({},{name:"Tom",age:21},{name:"Jerry",sex:"Boy"})

    那么合并后的结果

    result={name:"Jerry",age:21,sex:"Boy"}

    也就是说后面的参数如果和前面的参数存在相同的名称，那么后面的会覆盖前面的参数值。

    • 扩展方法
    $.extend({
   hello:function(){alert('hello');}
});
//使用
$.hello();

  2. 对象级别（$.fn.extend）

    从字面理解，这个拓展的是jQuery.fn的方法。jQuery.fn.extend拓展的是jQuery对象（原型的）的方法啊！对象是啥？就是类的实例化嘛。要使用这个扩展了的方法，必须在实例化的对象上使用才行。jQuery.fn.extend = jQuery.prototype.extend

    //定义了一个方法：
$.fn.tooltip = function(){
   alert("1");
}
//那么使用
$.tooltip() //错误的
$(".ID").tooltip()  //正确

return this.each()函数

解释这个函数，需要了解jQuery的链式操作(好处：节省代码量，减少DOM操作，代码看起来更优雅)，顾名思义，就是可以不停地进行操作。调用方法后返回对象，就可以继续操作了。

$("li").addClass("name").sibling().removeClass("name");

jQuery.fn.test = function(){ 
   //这里面的this为jquery对象，而不是dom对象 
   this.css("background","#ff0");

   //遍历匹配的元素，此处的this表示为jquery对象，而不是dom对象 
   return this.each(function(){ 
        //提示当前对象的dom节点名称,这里的this关键字都指向一个不同的DOM元素（每次都是一个不同的匹配元素）。
   	alert("this"+this+this.innerHTML);  
  }); 
};

()()的好处

眼尖的同学看到了，封装函数的第一步的代码有2个括号。

(function($){
        //代码
})(jQuery);

它有什么用呢？它可以立即执行一段代码，并把结果赋给变量；打破我们常规的先写函数，再调用的流程，简化书写。另外定义函数，然后马上执行它，便于保护它里面的变量，不被外边作用域的干扰。

function foo() {    
  var a = "aaa";    
  (function() {  
    var b = "bbb";  
    console.log(b); // bbb    
    console.log(a+"---"); //可以读取外面的变量a
  })();  
  console.log(b+"===");  //undefined  
}  
foo();
